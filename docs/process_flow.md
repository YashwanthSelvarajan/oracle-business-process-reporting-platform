# Process Flow

## As-Is Process
Business teams manage process requests through email and spreadsheets. Approval status is difficult to track, reporting is manual, and KPI visibility is limited.

## To-Be Process

1. Requestor submits a business process request.
2. System stores request details in Oracle tables.
3. Approver reviews the request.
4. PL/SQL package updates workflow status.
5. Reporting views refresh KPI dashboard logic.
6. Analyst reviews SLA status and cycle time trends.

## Business Value

- Reduces manual tracking
- Improves reporting consistency
- Supports audit-friendly approval history
- Gives stakeholders centralized visibility
