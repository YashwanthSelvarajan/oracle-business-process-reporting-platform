# Oracle Cloud-Based Business Process & Reporting Automation Platform

## Project Overview

This project simulates an Oracle APEX-based enterprise reporting and workflow automation platform. It is designed to demonstrate Oracle SQL, PL/SQL, business process tracking, approval workflows, KPI reporting, and packaged application support skills.

The project is portfolio-focused and aligns with Oracle Package Specialist and business technology consulting responsibilities such as requirement analysis, process mapping, configuration support, testing, reporting, and production support.

## Business Problem

Organizations often manage approval requests, reporting tasks, and operational processes across spreadsheets and disconnected systems. This creates delays, inconsistent reporting, limited visibility, and manual follow-up work.

## Solution

This platform centralizes process requests, approval tracking, KPI reporting, and workflow status monitoring using an Oracle database structure and PL/SQL business logic.

## How to Run

1. Open Oracle SQL Developer.
2. Connect to your Oracle Database or Oracle Cloud ATP instance.
3. Run the SQL scripts in this order:
   - `01_schema.sql`
   - `02_sample_data.sql`
   - `03_reporting_views.sql`
   - `04_workflow_package.sql`
4. Test the workflow package using the sample procedure calls in `04_workflow_package.sql`.
5. Review reporting views to simulate dashboard outputs.
